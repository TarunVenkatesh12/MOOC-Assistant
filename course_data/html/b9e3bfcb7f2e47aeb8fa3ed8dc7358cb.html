S'assurer que le serveur répond correctement

Dans le code précédent, nous sommes sûrs que les couches Wifi et IP sont fonctionnelles, il nous reste maintenant à tester que le serveur nous répond correctement. Nous viendrons pour cela tester deux choses :

  1. Le code HTTP renvoyé par le serveur. Ce code permet d'informer les clients (et donc, notre carte Galaxia) si la requête a fonctionné ou si celle-ci a échoué. En cas d'erreur, le code nous permettra même d'en savoir un peu plus sur la nature de l'erreur. Voici une liste des codes les plus courants sont :
    • 200 : succès de la requête ;
    • 301 et 302 : redirection, respectivement permanente et temporaire ;
    • 401 : utilisateur non authentifié ;
    • 403 : accès refusé ;
    • 404 : ressource non trouvée ;
    • 500, 502 et 503 : erreurs serveur ;
    • 504 : le serveur n'a pas répondu.
    ▶️ Pour une liste exhaustive des codes HTTP, vous pouvez vous reporter à cette fiche Wikipédia.
  2. Le contenu de la réponse du serveur. Même si la requête HTTP échoue, le serveur renverra la plupart du temps un texte explicatif sur l'erreur rencontrée. Par exemple :{"success":false,"message":"L'identifiant ou le mot de passe saisi est vide"} lorsque vous omettez votre clé d'API dans une requête à emoncms.fr.

Dans l'exemple ci-dessous, nous viendrons vérifier après chaque envoi de données le code HTTP et le contenu textuel reçus du serveur. Si ces tests sont concluants alors on allumera la LED RGB de la Galaxia en vert pendant un court instant sinon la LED passera en rouge.

⚠️ Attention : N'oubliez pas de modifier votre clé d'API dans le programme suivant.
