Activité : Le thermomètre endormi

Niveau de difficulté : ☆☆ (moyen)

Contexte

Marie souhaite connaître la température qui règne dans le réfrigérateur du wagon bar qui n'est malheureusement plus alimenté suite à une défaillance technique. Elle souhaite utiliser le capteur de température de la Galaxia pour assurer des mesures régulières en gardant une consommation électrique la plus basse possible car la Galaxia sera alimentée sur batterie !

Consignes

À partir des concepts abordés cette semaine, écrivez un programme avec l'interface Wokwi ci-dessous permettant de :

  1. Lire le fichier appelé data.txt et convertir la valeur contenue dans ce fichier en nombre flottant. Cette valeur sera stockée dans une variable appelée previous_temp.
  2. Comparer previous_temp avec la température courante obtenue avec la fonction temperature() issue du module thingz (note : ce module est déjà importé dans le programme ci-dessous).
  3. Si la température courante diffère de plus de 1°C (en plus ou en moins) à la valeur de previous_temp alors vous devrez envoyer cette valeur grâce à une requête HTTP POST vers l'URL https://emoncms.fr/input/post (en utilisant le module requests).
  4. À l'issue de ces étapes, vous devrez enregistrer la température courante dans le fichier  data.txt puis endormir la Galaxia pour une durée de 10 secondes grâce à la fonction machine.deepsleep().

Indices
    1. Lorsque vous lancerez pour la première fois le programme, le fichier data.txt n'existera pas et votre programme lèvera une erreur. Pas d'inquiétude, vous pouvez compter sur un bloc try except pour vous sortir d'affaire.
    2. Une manière élégante pour comparer la valeur courante de la température et previous_temp (et voir si cette différence est à ±1°C) pourrait être d'utiliser la fonction abs().
