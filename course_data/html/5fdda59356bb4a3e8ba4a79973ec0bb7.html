Dans les semaines précédentes, nous avons vu comment capter une donnée depuis un capteur physique, comment stocker cette donnée dans une variable et dans un dictionnaire, il est temps maintenant d'envoyer cette donnée structurée à un serveur sur internet. Pour cela, nous avons choisi d'utiliser une application web appelée Emoncms qui jouera le rôle de serveur tandis que notre carte Galaxia avec son module ESP jouera le rôle de client.

Pour rappel, l'architecture d'internet est appelée client/serveur où nous avons d'un côté des clients (un programme MicroPython par exemple ou un navigateur web) qui envoient des requêtes pour accéder aux ressources d'un serveur, qui traite ces requêtes et renvoie les réponses appropriées.

Emoncms, c'est quoi ça ?

Le serveur Emoncms est une application web libre qui permet de stocker, traiter et visualiser des données envoyées par des clients. Il a été développé dans le cadre d'un projet de gestion de données énergétiques mais permet de traiter aujourd'hui toute sorte de données !

Le site emoncms.fr est une instance de ce logiciel (hébergé sur un serveur de Baptiste), sur lequel nous vous proposons de travailler. Mais avant ça, un peu de vocabulaire propre à Emoncms :

L'API

L'API (ou interface de programmation en bon français) de emoncms vous permet  d'échanger des données depuis un client. Cette API consiste en requêtes HTTP :

  • Avec un verbe HTTP :
    • Le verbe GET  lorsque l'on voudra lire une donnée depuis emoncms.
    • Le verbe POST pour écrire une donnée.
  • Avec un chemin (une url) donné. Exemples :
    • GET emoncms.fr/feed/timevalue.json?id=76 : permet de récupérer un JSON avec la dernière valeur de température dans le bureau de Baptiste ainsi que son horodatage.
    • POST emoncms.fr/input/post?node=galaxia&fulljson={"temperature":20.1}&apikey=<clé_api> permet d'envoyer une valeur appelée temperature portant la valeur 20,1 envoyé par le client appelé galaxia.
  • Avec une clé d'API (ou API key en anglais) qui vous permettra de vous identifier sur le serveur emoncms (voir le tutoriel de création de compte pour savoir comment obtenir votre clé d'API).


Les sources (inputs)

Les sources dans emoncms sont créées automatiquement lors de l'envoi d'une requête POST valide dans le chemin emoncms.fr/input/post. La donnée envoyée par votre client s'affichera automatiquement sur la page Sources mais seule la données la plus récente est conservée. Chaque donnée dispose d'un nom (temperature dans l'exemple ci-dessus) et est associé à un client, appelé aussi nœud (node) dans emoncms.


Les flux (feeds)

Les sources que vous souhaitez conserver devront être stockées sous forme de séries de données temporelles, ou plus simplement séries temporelles (timeseries), qui pourront être ensuite visualisées sur le serveur. Une série temporelle est appelé flux dans emoncms et revient simplement à associé une heure/date (horodatage) à chaque donnée stockée sur le serveur.

Il pourra parfois être utile de faire des opérations mathématiques sur les sources avant de les stocker dans un flux. Pour cela, nous pourrons utiliser des traitements qui sont sont des opérations de transformation qui seront faites avant leur stockage dans un flux. On peut utiliser un traitement pour calibrer, ajouter, soustraire, multiplier, convertir, agréger... une source avant que celle-ci ne soit stocké dans un flux.


Les graphiques (graphs)

Les graphiques sont un moyen facile et rapide de visualiser les flux stockés dans emoncms. Ils disposent de nombreuses options de visualisation et de comparaison de flux. Les graphiques peuvent être inclus dans les tableaux de bord décrits ci-dessous.

Les visualisations (visualization)

Les visualisations offrent un éventail plus large de visualisations comme l'affichage temps-réel des données (RealTime), la comparaison de flux (Compare) ou encore l'affichage de nombreux flux avec MultiGraph.



Tableaux de bord (Dashboards)

Permet de créer des tableaux de bord personnalisés avec une interface glisser-déposer, en utilisant plusieurs élements (widgets) et graphiques et visualisations mentionnés ci-dessus . Un tableau de bord peut être rendu public et partagé avec un lien accessible sur le web.
