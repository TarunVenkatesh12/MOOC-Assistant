Activité : Horodatage ferroviaire

L'activité ci-dessous est notée et dispose de plusieurs versions avec des difficultés croissantes :

    ⭐️⭐️ : Facile
    ⭐️⭐️⭐️ : Moyen
    ⭐️⭐️⭐️⭐️ : Difficile

Pour obtenir la note maximale, vous devez valider les trois versions !

Contexte

Baptiste souhaite suivre l'activité de la locomotive en bois de sa fille dans le temps. Cette locomotive embarque une carte Galaxia qui stockera et affichera les derniers événements horodatés (=précédés de l'heure à laquelle l'événement a eu lieu).

Consignes

À partir des concepts abordés cette semaine, écrivez un programme avec l'interface FranceIOI ci-dessous qui viendra stocker et afficher les événements horodatés :

  • Événement "power-up" : Lorsque l'on appuie sur le bouton A, connectez la Galaxia à un réseau wifi en mode station.
  • Événement "move-forward" : Lorsque l'on appuie sur le bouton tactile « haut », la LED RGB devra s'allumer en blanc avec ses trois valeurs RGB à 255.
  • Événement "reverse" : Lorsque l'on appuie sur le bouton tactile « bas », la LED RGB devra s'allumer en rouge (Rouge = 255, Vert = 0, Bleu = 0).
  • Événement "whistle" : Lorsque l'on appuie sur le bouton tactile « droite », le buzzer (branché sur le port Grove 1) devra émettre la note A5 (880Hz) pendant 500 millisecondes.
  • Événement "stop" : Lorsque l'on appuie sur le bouton B, la LED RGB devra s'éteindre et le wifi devra être déconnecté.

Ces événements devront être capturés indéfiniment grâce à une boucle Répéter indéfiniment. Chacun des événements devra être stocké dans une liste de dictionnaires selon la structure suivante :

events = [
{"timestamp": "1730708895", "event": "power-up"},
{"timestamp": "1730708900", "event": "move-forward"},
{"timestamp": "1730708922", "event": "stop"},
]

L'élément timestamp devra correspondre à l'heure unix (nombre de secondes écoulées depuis 1ᵉʳ janvier 1970 00:00:00) récupérée grâce à une requête HTTP GET vers l'URL https://api.emoncms.fr/time (en utilisant le module requests).
