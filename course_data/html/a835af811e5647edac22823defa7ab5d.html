Activité : Le tunnel

L'activité ci-dessous est notée et dispose de plusieurs versions avec des difficultés croissantes :

    ⭐️⭐️ : Facile
    ⭐️⭐️⭐️ : Moyen

Pour obtenir la note maximale, vous devez valider les deux versions !

Contexte

Jean souhaite suivre la température dans la locomotive depuis son téléphone connecté à Internet. Il compte utiliser la carte Galaxia pour envoyer la température sur internet toutes les deux secondes. Mais Jean se demande comment ne pas perdre ses données lorsque le train passe sous un tunnel. À vous de l'aider !

Consignes

À partir des concepts abordés dans le MOOC, écrivez un programme avec l'interface FranceIOI ci-dessous qui viendra :

    1. Connecter la Galaxia à un réseau wifi en mode station.
    2. Lire la valeur du capteur de température.
    3. S'assurer que la Galaxia est bien connectée. Deux possibilités devront être traitées :
      • Si la connexion est maintenue, vous devrez envoyer ces données en utilisant une requête HTTP POST vers le site emoncms.fr (en utilisant le module additionnel requests).
      • Si la locomotive passe sous un tunnel et que la connexion est perdue, vous devrez stocker la température dans une liste. Lorsque la connexion est rétablie, vous devrez envoyer toutes les données stockées dans la liste en envoyant  une requête HTTP POST vers l'url emoncms.fr/input/bulk (en utilisant le module additionnel requests). Nous vous invitons à consulter l'aide de l'API Sources d'Emoncms pour en savoir plus sur l'envoi de plusieurs données horodatées.
    4. Vérifier que les données sont bien reçues en vérifiant le code HTTP et la réponse du serveur.
    5. Attendez 10 secondes avant de revenir à l'étape 3 et cela indéfiniment.
