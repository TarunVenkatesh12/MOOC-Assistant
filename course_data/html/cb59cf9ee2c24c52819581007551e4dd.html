Étape 3 : TCP et HTTP

Nous arrivons à la dernière étape et probablement la plus intéressante car nous avons maintenant nos couches Wifi et IP qui sont fonctionnelles et nous allons pouvoir nous connecter à un serveur web. Pour cette dernière étape, notre module ESP jouera le rôle de client et nous allons interrogé un ordinateur situé sur Internet, appelé un serveur, pour obtenir une ressource ! C'est parti :

Dans le code qui suit, nous chercherons à interroger un serveur web (worldtimeapi.org) qui fournit à qui le demande gentiment les heures de tous les pays du monde 🙂. Nous reprendrons donc le code des deux sections précédentes pour se connecter au wifi puis avoir une connectivité IP. Nous viendrons ensuite établir une connexion TCP entre l'ESP et le serveur situé sur Internet. Le protocole TCP s'assurera que la requête sera bien relayée par la box wifi et arrive bien jusqu'au serveur sur internet et que la réponse du serveur revienne bien jusqu'à l'ESP. Comme le serveur worldtimeapi.org parle le protocole HTTP, notre module ESP parlera lui aussi HTTP grâce au module additionnel appelé urequests (renommé simplement requests dans les dernières versions de micropython). Ce module est installé par défaut dans les interfaces Vittascience   et FranceIOI   tandis que vous devrez l'installer par vous même sur votre carte Galaxia en suivant le tutoriel en bas de cette page.
