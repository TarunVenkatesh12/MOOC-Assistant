LE JSON

En manipulant des objets connectés nous allons forcément manipuler… de la donnée ! Comme vous pouvez vous y attendre, cette donnée utilisera la magie internet pour naviguer dans les méandres du réseau (nous verrons cette partie en détail plus tard). Les objets connectés ayant besoin d’économiser leur énergie au maximum, nous allons devoir nous tourner vers un format de données simple, léger et compact. C’est ici qu’intervient JSON !

Un langage simple et efficace

JSON, prononcé “jisone” en français ou jay-zon avec l'accent anglais - n'est pas un personnage de films d'horreur, mais l'acronyme de Java Script Object Notation. C'est un format qui a été à l'origine conçu pour transporter des données pour les pages Web. Mais, à l'inverse du tueur en série, JSON étant très tolérant, il a été très vite utilisé pour structurer d'autres informations et est très populaire pour l'internet des objets, car, en plus, il est compact et simple à utiliser.

Structurer des données est quelque chose que nous faisons au quotidien, par exemple en écrivant ces quelques lignes, je sépare et organise les mots pour les rendre digestes : sinonçavientviteduràlire !

Pour les données numériques c'est la même chose. JSON définit deux types de données de base :

  • les chaînes de caractères qui sont entre "guillemets"
  • les nombres qui eux ne sont pas entre guillemets et qui contiennent principalement des chiffres. Par exemple "123" est une chaîne de caractères et -100 est un nombre.

Quelques exemples pour mieux comprendre

Prenons un ensemble, imaginons que l’on veuille recueillir des informations concernant la météo du jour. Commençons par une version simplifiée : on veut connaître la température, l’humidité et la pression atmosphérique du jour.

Voici à quoi ça ressemblerait en JSON :

{"temperature": 25, "humidite": 65, "pression_atmo" : 1012}

Nous venons de créer une liste de paires clé/valeur. Cette liste est délimitée par des accolades {}, et remplie de paires composées d'une clé qui est une chaîne de caractère entre guillemets suivie de deux points et enfin d’une valeur qui peut être un nombre ou une autre chaîne de caractère.

{"temp_min_max" : [12,28]}

JSON permet aussi de faire des tableaux, ils sont délimités par des crochets droits et les éléments sont séparés par des virgules.

Tout ça c’est bien beau mais ça manque encore de détails ! Dans ce code, nous n’indiquons pas la localisation de la météo demandée, c’est pourtant essentiel !

Un exemple météo

Voici un exemple plus détaillé qui donne la météo du jour à Lyon :

{
"Lyon": {
"meteo": {
"theme": "soleil",
"temperature": 25.31,
"ressentie": 25.59,
"temp_min_max": [
12,
28
],
"pression_atmo": 1012,
"humidite": 65,
"vigilance_meteo": "verte"
},
"vent": {
"vitesse": 0.33,
"angle": 125,
"rafales": 0.67
},
"station": {
"id": 2996943,
"name": "Lyon 7"
}
}
}

Ça commence à faire beaucoup d’informations ! On retrouve nos listes et tableaux, sauf que cette fois la liste contient d’autres listes et des tableaux !

Vous comprenez maintenant l’intérêt de donner une structure à l’information. Imaginez maintenant que cette liste fait partie d’une autre liste plus grande qui inclut toutes les stations françaises puis encore une autre qui compte toutes les stations mondiales ! L’intérêt d’avoir un classement cohérent devient vital.
